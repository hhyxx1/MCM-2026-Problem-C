## 第一部分: 符号定义表

### 1.1 基础数据符号

| 符号 | 含义 | 取值范围 | 备注 |
|------|------|----------|------|
| i | 选手索引 | 整数 | 每赛季内唯一标识 |
| t | 周次/时间 | 1, 2, ..., T | T为该赛季总周数 |
| s | 赛季编号 | 1-34 | 覆盖所有已播出赛季 |
| 𝒥(i,t) | 评委得分百分比 | 0-100 | 统一标准化后的评委分数 |
| f(i,t) | 粉丝投票份额估计值 | [0, 1] | 满足 Σ_i f(i,t) = 1 |
| ℰ_t | 第t周淘汰选手集合 | 选手子集 | 大小为k(本周淘汰人数) |
| n | 某周参赛选手总数 | 正整数 | 随周数递减 |
| k | 某周淘汰人数 | 正整数 | 通常为1或2 |

### 1.2 排名与偏差指标

| 符号 | 含义 | 计算方式 | 解释 |
|------|------|----------|------|
| R^𝒥_i | 选手i的评委排名 | 按𝒥降序排列 | 1=最高分 |
| R^*_i | 选手i的最终名次 | 实际比赛结果 | 1=冠军 |
| R^f_i | 选手i的粉丝排名 | 按f估计值降序排列 | 1=最受欢迎 |
| Δ_i | 人气偏差指数 | R^𝒥_i - R^*_i | 衡量粉丝拯救效应 |
| r^𝒥(i,t) | 第t周评委排名 | 周内𝒥降序 | 每周重新计算 |
| r^f(i,t) | 第t周粉丝排名 | 周内f(i,t)降序 | 基于估计值 |

### 1.3 聚合得分符号

| 符号 | 含义 | 公式形式 | 用途 |
|------|------|----------|------|
| S^r(i,t) | 基于排名的综合得分 | α·r^𝒥 + (1-α)·r^f | 排名加权法 |
| S^p(i,t) | 基于百分比的综合得分 | α·𝒥 + (1-α)·F | 百分比加权法(现行) |
| F(i,t) | 粉丝得分百分比 | [f(i,t) / max f] × 100 | 归一化到0-100 |
| α | 评委权重(静态) | [0, 1] | 默认0.5(50-50分配) |
| α(t) | 评委权重(动态) | 随时间变化 | 动态规则中使用 |

### 1.4 贝叶斯推断符号

| 符号 | 含义 | 计算方式 | 说明 |
|------|------|----------|------|
| f̄(i,t) | 粉丝投票份额后验均值 | MCMC样本均值 | 点估计 |
| f̃(i,t) | 粉丝投票份额后验中位数 | MCMC样本中位数 | 稳健估计 |
| f_L(i,t) | 95%可信区间下界 | 2.5%分位数 | 不确定性度量 |
| f_U(i,t) | 95%可信区间上界 | 97.5%分位数 | 不确定性度量 |
| W(i,t) | 可信区间宽度 | f_U(i,t) - f_L(i,t) | 估计精度指标 |
| p_t | 后验一致性概率 | P(ℰ_t为Bottom-k \| 数据) | 模型验证指标 |
| N | MCMC样本数 | 2000-5000 | 采样规模 |

### 1.5 公平性与参与度指标

| 符号 | 含义 | 公式 | 范围 | 解释 |
|------|------|------|------|------|
| ℱ | 粉丝偏爱指数 | ρ_Spearman(R^*, R^f) | [-1, 1] | 越高越"粉丝导向" |
| ℳ | 择优性指数 | ρ_Spearman(R^*, R^𝒥) | [-1, 1] | 越高越"精英主义" |
| ℬ | 平衡得分 | 2ℳℱ/(ℳ+ℱ) | [0, 1] | 调和平均数 |
| 𝒟 | 分歧分数 | 1 - ρ(R^𝒥, R^*) | [0, 2] | 评委-结果不一致度 |
| ℰ | 粉丝弹性 | P(淘汰变化 \| 投票扰动) | [0, 1] | 规则稳定性指标 |

### 1.6 动态规则参数

| 符号 | 含义 | 典型值 | 说明 |
|------|------|--------|------|
| α₀ | 初始评委权重 | 0.5 | 第0周的α值 |
| δ | 周增量 | 0.03 | 每周α的增加量 |
| ℓ | 对数强度 | 0.5 | log变换的权重 |
| c | Sigmoid陡度 | 6 | S型曲线的陡峭程度 |
| α_min | 权重下界 | 0.30 | Sigmoid函数的渐近值 |
| α_max | 权重上界 | 0.75 | Sigmoid函数的渐近值 |

### 1.7 回归模型符号

| 符号 | 含义 | 模型中的角色 |
|------|------|--------------|
| β_t | 周次效应系数 | 固定效应(时间趋势) |
| β_{t²} | 周次二次项系数 | 固定效应(非线性趋势) |
| β_s | 赛季效应系数 | 固定效应(年份差异) |
| u_p | 职业舞者随机效应 | 个体异质性 |
| u_c | 名人随机效应 | 个体天赋/人气 |
| e_t | 周随机效应 | 时间随机扰动 |
| ε | 残差项 | 未解释的随机误差 |
| ρ_I | 组内相关系数 | σ²_between / (σ²_between + σ²_within) |
| h | 技能溢出效应 | 𝒥对log(f)的影响系数 |

### 1.8 Pareto优化符号

| 符号 | 含义 | 几何意义 |
|------|------|----------|
| 𝒥(规则) | 规则的择优性 | Pareto空间的x坐标 |
| ℱ(规则) | 规则的参与性 | Pareto空间的y坐标 |
| d^* | 膝点距离 | 到对角线的欧氏距离 |
| 𝒥̃ | 归一化择优性 | [0,1]标准化后的𝒥 |
| ℱ̃ | 归一化参与性 | [0,1]标准化后的ℱ |

## 第二部分: 核心数学公式

### 公式1: 评委得分标准化 (Judge Score Standardization)

**公式表达:**
```
𝒥(i,t) = [Score_raw(i,t) / Score_max(t)] × 100
```

**含义**: 将原始评委分数(30分制或40分制)统一标准化到0-100百分比量纲。

**解决的问题**:
- 跨赛季可比性: 早期赛季用3评委(30分满分),后期用4评委(40分满分)
- 消除量纲差异,使得不同时期的分数可以直接比较

**结果反映的特征**:
- 𝒥 = 100表示该周满分表现
- 𝒥 < 70通常意味着处于淘汰风险区

**对应图表**:
- [cleaned_outputs/global_scan_heatmap.png](cleaned_outputs/global_scan_heatmap.png) - 热力图中的颜色基准

**对解决问题的帮助**: 为后续所有分析提供统一的评委分数基准,是整个模型的数据基础。

### 公式2: 人气偏差指数 (Popularity Bias Index)

**公式表达:**
```
Δ_i = R^𝒥_i - R^*_i
```

**含义**: 衡量选手的最终排名相对于评委给出的能力排名的偏离程度。

**解决的问题**:
- 识别"粉丝宠儿": Δ_i > 0表示粉丝把排名低的选手"拉上来"
- 识别"评委宠儿": Δ_i < 0表示评委高分但粉丝不买账
- 量化粉丝投票对结果的扭曲程度

**结果反映的特征**:
- Δ = +6 (Bobby Bones, S27): 评委排第7但夺冠,极端粉丝效应
- Δ = -3: 评委给高分但早早淘汰,技术型选手不讨喜
- Δ接近0: 评委和粉丝意见一致

**图表体现**:
- [cleaned_outputs/pbi_analysis.png](cleaned_outputs/pbi_analysis.png) - Δ分布直方图,极值标注
- [cleaned_outputs/pbi_vs_score.png](cleaned_outputs/pbi_vs_score.png) - Δ与𝒥的散点图
- [cleaned_outputs/feature_engineering/pbi_by_industry.png](cleaned_outputs/feature_engineering/pbi_by_industry.png) - 不同职业背景的Δ对比

**对解决问题的帮助**:
- 为设计"评委保护机制"提供量化依据
- 证明现行规则存在显著的粉丝操纵风险

### 公式3: 基于排名的聚合 (Rank-based Aggregation)

**公式表达:**
```
S^r(i,t) = α · r^𝒥(i,t) + (1-α) · r^f(i,t)
其中: r^𝒥 = rank(-𝒥), r^f = rank(-f)  (降序排名)
```

**典型参数**: α = 0.5 (50-50均等权重)

**含义**: 将评委排名和粉丝排名取平均,得分越低越安全。

**解决的问题**:
- 极端值稳健性: 排名对异常高/低的投票不敏感
- 序数公平: 只关心相对位置,不受绝对值尺度影响

**结果反映的特征**:
- ℳ_rank = 0.7274 (评委偏爱度高)
- ℱ_rank = 0.7670 (粉丝偏爱度也高)
- 两者相对平衡,但略偏向粉丝

**图表体现**:
- [cleaned_outputs/phase3_simulator/ffi_jfi_tradeoff.png](cleaned_outputs/phase3_simulator/ffi_jfi_tradeoff.png) - 蓝色圆点簇(Rank方法在𝒥-ℱ平面的位置)
- [cleaned_outputs/phase3_simulator/jfi_comparison.png](cleaned_outputs/phase3_simulator/jfi_comparison.png) - 蓝色柱(Rank方法的ℳ)
- [cleaned_outputs/pareto_optimization.png](cleaned_outputs/pareto_optimization.png) - Pareto前沿上的蓝色点

**对解决问题的帮助**:
- 证明Rank方法比Pct方法更择优(ℳ更高)
- 为推荐规则提供基准对比

### 公式4: 基于百分比的聚合 (Percentage-based Aggregation)

**公式表达:**
```
S^p(i,t) = α · 𝒥(i,t) + (1-α) · F(i,t)
其中: F(i,t) = [f(i,t) / max_j f(j,t)] × 100
```

**典型参数**: α = 0.5 (当前DWTS使用的规则)

**含义**: 直接将评委百分比分数和粉丝百分比投票加权平均,得分越高越安全。

**解决的问题**:
- 与电视节目的实际操作对齐
- 观众易理解(分数相加)

**结果反映的特征**:
- ℳ_pct = 0.3735 (评委偏爱度**极低**,仅为Rank方法的51%)
- ℱ_pct = 0.7882 (粉丝偏爱度略高于Rank)
- 严重偏向粉丝,评委意见被稀释

**图表体现**:
- [cleaned_outputs/phase3_simulator/ffi_jfi_tradeoff.png](cleaned_outputs/phase3_simulator/ffi_jfi_tradeoff.png) - 红色方块簇(Pct方法显著偏右下)
- [cleaned_outputs/phase3_simulator/jfi_comparison.png](cleaned_outputs/phase3_simulator/jfi_comparison.png) - 红色柱(Pct的ℳ明显低于Rank)
- [cleaned_outputs/pareto_optimization.png](cleaned_outputs/pareto_optimization.png) - 六边形标记的"Current"点(远离Pareto前沿)

**对解决问题的帮助**:
- 暴露现行规则的核心缺陷: 不公平(低ℳ)
- 为改革提供紧迫性论证

### 公式5: 贝叶斯逆推断约束 (Bayesian Inverse Inference Constraints)

**公式表达:**
```
约束条件:
(1) Σ_i f(i,t) = 1  (投票份额之和为1)
(2) f(i,t) ≥ 0  (非负)
(3) ∀e ∈ ℰ_t, ∀s ∉ ℰ_t: Score(s,t) > Score(e,t)  (淘汰者必在Bottom-k)
```

**含义**: 在已知淘汰结果的情况下,反推粉丝投票分布必须满足的数学约束。

**解决的问题**:
- 核心难题: 粉丝投票份额从未公开,需要从淘汰模式中推断
- 不确定性量化: 提供可信区间而非单一猜测值

**结果反映的特征**:
- 平均W = 0.288 (约29%的不确定性)
- 精确匹配率 = 95.6% (模型高度可靠)
- 早周不确定性高(选手多),晚周收窄(选手少)

**图表体现**:

- [cleaned_outputs/bayesian_inference/ci_width_distribution.png](cleaned_outputs/bayesian_inference/ci_width_distribution.png) - W分布直方图
- [cleaned_outputs/bayesian_inference/ci_width_by_week.png](cleaned_outputs/bayesian_inference/ci_width_by_week.png) - W随周次变化(倒U型)

**对解决问题的帮助**:
- 为后续模拟提供必需的f(i,t)估计值
- 验证了逆推方法的可行性(高精确匹配率)

### 公式6: 后验一致性概率 (Posterior Consistency Probability)

**公式表达:**
```
p_t = (1/N) Σ_{j=1}^N 𝟙[ℰ_t = Bottom-k under sample_j]
```
其中𝟙[·]为指示函数

**含义**: 在MCMC后验样本中,有多大比例能够正确预测本周淘汰的k个选手。

**解决的问题**:
- 模型验证: 检验推断出的f(i,t)是否与实际淘汰模式一致
- 识别异常周: p_t过低可能表示有意外事件(如选手退赛)

**结果反映的特征**:
- 全局平均 p̄ = 0.651 (65.1%的一致性)
- Jaccard指数 = 0.960 (集合重叠度极高)
- 精确匹配率 = 95.6% (接近完美)

**对解决问题的帮助**:
- 证明贝叶斯推断方法高度可靠
- 为置信区间的解释提供支持

### 公式7: 可信区间宽度 (Credible Interval Width, 不确定性度量)

**公式表达:**
```
W(i,t) = f_U(i,t) - f_L(i,t)
其中 f_L = q_{2.5%}(f), f_U = q_{97.5%}(f)
```

**含义**: 95%可信区间的宽度,数值越大表示对该选手投票份额的估计越不确定。

**解决的问题**:
- 量化估计精度: 哪些选手/周的推断更可靠
- 识别信息匮乏区: W宽可能因淘汰约束弱

**结果反映的特征**:
- 平均W = 0.288
- 最窄W ≈ 0.05 (晚期周,选手少,约束强)
- 最宽W ≈ 0.60 (早期周,选手多,信息弱)

- [cleaned_outputs/patch3_certainty/ci_width_distribution.png](cleaned_outputs/patch3_certainty/ci_width_distribution.png) - W频率分布

**对解决问题的帮助**:
- 指导使用估计值时的谨慎程度
- 识别模型的弱点(早期周不确定性高)

### 公式8: 择优性目标 (Meritocracy Objective)

**公式表达:**
```
ℳ = ρ_Spearman(R^*, R^𝒥)
```
其中ρ_Spearman为Spearman秩相关系数

**含义**: 最终排名与评委给出的技术排名的一致性,衡量"高手该得奖"的程度。

**解决的问题**:
- 定义"公平"的量化标准
- 在Pareto优化中作为y轴(或x轴)目标之一

**结果反映的特征**:
- Rank方法: ℳ = 0.665 (较高择优性)
- Pct方法: ℳ = 0.454 (择优性差,仅68%的Rank)
- 推荐规则: ℳ = 0.665 (保持高择优性)

### 公式9: 参与性目标 (Engagement Objective)

**公式表达:**
```
ℱ = ρ_Spearman(R^*, R^f)
```

**含义**: 最终排名与粉丝支持度排名的一致性,衡量"观众喜欢谁就留下谁"的程度。

**解决的问题**:
- 定义"参与度"的量化标准
- 防止规则过度精英化导致观众流失

**结果反映的特征**:
- Rank方法: ℱ = 0.704 (略低于Pct)
- Pct方法: ℱ = 0.691 (虽高但牺牲了ℳ)
- 推荐规则: ℱ = 0.704 (保持高参与度)

### 公式10: Pareto膝点识别 (Pareto Knee Point Identification)

**公式表达:**
```
𝒥̃ = (𝒥 - 𝒥_min) / (𝒥_max - 𝒥_min)
ℱ̃ = (ℱ - ℱ_min) / (ℱ_max - ℱ_min)
d_i = |𝒥̃(i) + ℱ̃(i) - 1| / √2
膝点 = argmax_i d_i
```

**含义**: 在Pareto前沿上找到距离"理想对角线"(𝒥̃ + ℱ̃ = 1)最远的点,即平衡性最优的规则。

**解决的问题**:
- 多目标优化的决策难题: 前沿上有无数点,选哪个?
- 膝点是"性价比"最高的折中方案

**结果反映的特征**:
- Rank方法: d^* = 0.224 (显著膝点)
- Pct方法: d^* = 0.060 (接近直线,无明显膝点)
- 推荐规则: 位于全局Pareto前沿的膝点附近

### 公式11: 评委得分回归模型 (Judge Score Regression - Mixed Effects)

**公式表达:**
```
𝒥(i,t) = α + β_t·t + β_{t²}·t² + β_s·s
         + u_p[partner_i] + u_c[i] + e_t + ε
```
其中:
- β为固定效应系数
- u为随机效应
- e_t为周随机效应
- ε为残差

**含义**: 将评委得分分解为时间趋势、赛季效应、职业舞者效应、名人效应等多个成分。

**解决的问题**:

- 效应分离: 职业舞者vs名人的相对贡献

**结果反映的特征**:
- R² = 0.352 (模型解释35.2%的方差)
- 职业舞者效应: ICC = 28.6% (最大单一来源)
- 名人效应: ICC = 52.6% (天赋+人设)
- 赛季效应: 3.4% (评分标准变化小)

### 公式13: 方差分解 (Variance Decomposition - ICC)

**公式表达:**

- σ²_within: 组内方差(同一舞者不同场次)

**含义**: 组内相关系数,表示总方差中有多少比例可归因于"组别"因素。

**解决的问题**:

### 公式14: 粉丝弹性 (Fan Vote Elasticity)

**公式表达:**
```
ℰ = (1/N_sim) Σ_{j=1}^{N_sim} 𝟙[Ê_t^perturbed ≠ ℰ_t^original]
其中: f_perturbed(i,t) = f(i,t) + 𝒩(0, σ²_noise)
典型σ_noise = 0.03
```

**含义**: 在粉丝投票上加入小幅随机扰动后,有多大比例的模拟会改变淘汰结果。弹性越高,规则越不稳定。

**解决的问题**:
- 规则稳健性检验: 小波动是否导致大变化?
- 识别"脆弱规则"

**结果反映的特征**:
- Rank方法: ℰ = 0.1374 (13.74%的周会变化)
- Pct方法: ℰ = 0.1216 (12.16%)
- Rank方法略不稳定,但差异不大

### 公式15: 推荐动态加权规则 (Recommended Dynamic Weighting Rule)

**公式表达:**
```
S(i,t) = α(t) · 𝒥(i,t) + [1-α(t)] · [ℓ·log(1+F(i,t)) + (1-ℓ)·F(i,t)]

其中动态权重:
α(t) = {
  0.50                  if t ≤ 3  (早期: 50-50平衡)
  0.50 + 0.05(t-3)      if 4 ≤ t ≤ 7  (中期: 线性增加)
  0.70                  if t ≥ 8  (后期: 70-30偏评委)
}

或更一般的形式:
α(t) = min(α₀ + δ·t, α_max)
典型值: α₀=0.5, δ=0.03, α_max=0.8, ℓ=0.5
```

**含义**: 评委权重随比赛进程动态增加,早期重视粉丝参与(吸引观众),后期重视技术评判(公平竞争)。对粉丝得分使用对数变换压缩极端值。

**解决的问题**:
- 核心矛盾: 既要公平(高ℳ)又要参与(高ℱ)
- 阶段性需求: 早期需流量,决赛需公正

**结果反映的特征**:
- 预期ℳ = 0.665 (相比现行+49%)
- 预期ℱ = 0.704 (相比现行+1.9%)
- ℬ = 0.683 (调和平均达到最优)
- 争议度降低60-70%

**图表体现**:

## 附录C: 符号系统设计原则

### ✅ 完全避免的歧义符号
- ❌ φ, Φ (黄金比例/正态分布)
- ❌ γ (伽马分布)
- ❌ λ (特征值/正则化)
- ❌ κ (曲率/条件数)
- ❌ η (效率/黏度)
- ❌ τ (时间常数/Kendall's tau)
- ❌ ζ (Riemann zeta)
- ❌ ω (角频率)
- ❌ ν (自由度)
- ❌ π (圆周率)

### ✅ 最终采用的符号类别
1. **拉丁字母**: i, t, s, n, k, f, F, h, c, e, p, u, d, W, N, R, S, L, U
2. **标准希腊字母** (标准统计用法): α, β, δ, ε, ρ
3. **花体字母** (关键指标): 𝒥, ℱ, ℳ, ℬ, 𝒟, ℰ, ℓ
4. **修饰符**: 上标(^), 下标(_), 横线(̄), 波浪线(̃), 星号(*)

### ✅ 符号-含义对应原则
- **𝒥族** (Judge相关): 𝒥, ℳ, R^𝒥, r^𝒥
- **f/F族** (Fan相关): f, F, ℱ, R^f, r^f
- **α/β** (权重): α(评委), β=1-α(粉丝)
- **u/e** (随机效应): u_p(舞者), u_c(名人), e_t(周)
- **花体字母** (关键指标): 𝒥, ℱ, ℳ, ℬ, 𝒟, ℰ
- **Δ** (差值/偏差): 人气偏差指数
- **R/r** (排名): R(最终排名), r(周排名)
- **S** (得分): S^r(Rank法), S^p(Pct法)
